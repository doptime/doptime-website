## doptime 配置模板
:::

<Tabs> 
<TabItem value="docker-compose" label="docker-compose">
```yaml title="docker-compose.yml"
version: '3'
services:
  doptimeBackend:
    image:  yangkequn/your-doptime-app:latest
    ports:
      - "8001:80"  
    restart: always  # Automatically restart the container if it stops
    environment:
      - CONFIG_URL=https://gist.githubusercontent.com/doptime/39da1f98236be1148c3f009308aada03/raw/322faf52677f4c5b197df61f2febaffb0c7cc373/config.toml
    networks:
      - my-shared-network
networks:
  my-shared-network:
    external: true
```

## **说明**
 所有变量的含义约定和配置文件一致。它就是配置文件的http(s)版本。

## **其它补充**  
doptime 加载顺序是:
1. 加载配置文件config.toml  如果成功
2. 从（环境变量中的CONFIG_URL || config.toml中的 CONFIG_URL）加载配置, 如果成功，则：
   -  覆盖配置文件，旧的配置文件会被删除！url中的配置会成为下次启动的默认值。这是为了避免配置服务器宕机。  
   -  覆盖环境变量，环境变量配置被忽略  
 CONFIG_URL 会被每分钟检查一次，如果有变化则会重新加载配置。
