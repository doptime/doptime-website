## **用简洁的框架** 集合那些美妙的优点　
:::
- **简洁**：仅使用Redis操作数据。高并发，快响应。同时代码简洁。
- **简洁**：当你定义了一个函数，它就是一个可以被RPC调用的微服务。也同时可以通过http调用。
- **简洁**：当你定义了一个函数的入参和出参。就可以替代使用生成器， 免于一切学习和繁琐的数据多次定义。
- **简洁**：数据的增删查改基本靠REDIS语句，你都不需要写CURD。
- **简洁**：不再区分单体服务和微服务。你的代码，单体服务, 也是微服务。但你想从单体切换到多体时，它已经是了。
- **简洁**：你随时可以像把网线插拔到交换机一下，热插拔你的软件。
- **简洁**：你只需专注核心业务代码，用心流的方式

:::info 哪些成本可以一砍再砍？
## 成本的底线在哪里？如何可以到达成本底线？
:::
**成本压缩**：开发成本的第一个核心压缩是重复开发。每个大公司都提供了API接口。但你还需要把它用起来。这这个用起来的过程往往是一个重复性繁琐的工作。doptime的目标是把这个重复性工作压缩到几乎没有。  
**成本压缩**：随着人工智能的硬件和软件的发展，硬件的昂贵开销使得数据中心的硬件托管成为不可能。doptime认为我们需要基于二手硬件，和一个基于大宗交易的竞价模式，把所有必须得服务费用降到最低。而普通的用户只需要使用这种数据处理服务。用户租用云服务长期来看是不必要，因为云服务商的毛利真的很高。doptime 旨在完成这种api的大宗采购。所有的服务都应该是一种端到端的服务。
**成本压缩**：doptime 认为。用户只需要管理自己的数据。需要尽可能摆脱API。API是一个额外的成本和复杂度。而用户数据才是重要有意义的。

**成本压缩**: 在人类历史上一切商业模式中，只有亚马逊会员，山姆会员，京东会员这样的业务模式才能最大化用户价值和服务商价值的完全一致性。

**成本压缩**：
提供一种尽可能无法再压缩的数据操作，来降低AI编码的可靠性和成本。这是doptime的目标。
