## **DOPTIME 存在的目的是为了降低开发部署的一切阻力**
- doptime 是深度优化 代码简洁性、性能、成本的 后端框架。
- 使用doptime 框架免费。
- doptime 最终目标是涵盖常见应用的RPC，使得RPC价格低于本机调用价格，并通过这些通用RPC盈利。
    

:::tip 简洁的框架
### **简洁**： 集合美妙优点　
  DopTime 最核心想法是尽可能取消代码。对人更友好, 为AI代码生成高度优化
:::
- **简洁**：仅使用Redis操作数据。高并发，快响应。同时代码简洁。
- **简洁**：当你定义了一个函数，它就是一个可以被RPC、HTTP调用的微服务。
- **简洁**：数据的增删查改基本靠REDIS语句，你都不需要写CURD。
- **简洁**：不再区分单体服务和微服务。你的代码，单体服务, 也是微服务。但你想从单体切换到多体时，它已经是了。
- **简洁**：免于使用代码生成器， 免于数据多次定义。
- **简洁**：你只需专注核心业务代码，用心流的方式  

:::tip  性能
### **性能**：最大化并发与最小化延迟
:::
- **性能**：dragonflydb, 可以提供每秒2M+的读写性能。  
- **性能**：对本地数据的读写延迟对应redis缓存的延迟。它的性能大大高于mysql。基本都在1ms以内。  

:::tip 成本
### **砍掉大部分，按真实需求计费**：
:::

- **成本压缩**：砍掉重复开发  
删掉重复开发。许多公司都提供了API接口。但你还需要把API用起来。这依然是重复繁琐的工作。  
doptime的目标是把这个重复性工作压缩到几乎没有。    
使用公开开源的RPC有更高的质量和更低的风险。  
如果你封装了一个通用接口，你应该把它发布到doptime的公开平台上。这样其它人可以直接调用你的接口。避免重复的开发。  
[加入我们的社区来发布你的需求，和分享你的接口](./intro/doptime-community)


- **成本压缩**：使用低成本的RPC  
AI的快速发展，意味着API的实现成本在指数级降低。它的价值是快速贬值的。  
doptime 致力于这种贬值， 试图把服务费用降到单机调用成本的水平。  
我相信在AI时代，趋于成本的，可靠的api会成为最重要的基础设施，而且很难本地化。  
使用第三方API可以让用户按真实业务需求而非业务预期来付费。以实现最小化支出和最大化弹性。
