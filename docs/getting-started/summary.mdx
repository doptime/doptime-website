## **用简洁的方式**集合各种框架最美妙的优点
:::
- **简洁**：仅使用Redis操作数据。高并发，快响应。同时代码简洁。
- **简洁**：当你定义了一个函数，它就是一个可以被RPC调用的微服务。也同时可以通过http调用。
- **简洁**：当你定义了一个函数的入参和出参。就可以替代使用生成器， 免于一切学习和繁琐的数据多次定义。
- **简洁**：数据的增删查改基本靠REDIS语句，你都不需要写CURD。
- **简洁**：不再区分单体服务和微服务。你的代码，单体服务, 也是微服务。但你想从单体切换到多体时，它已经是了。
- **简洁**：你随时可以像把网线插拔到交换机一下，热插拔你的软件。
- **简洁**：你只需专注核心业务代码，用心流的方式
